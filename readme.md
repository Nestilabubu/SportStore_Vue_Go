## О проекте

**SportShop** — это полнофункциональный интернет-магазин спортивных костюмов. Представляет собой веб-приложение, где пользователи могут:
- просматривать каталог товаров с фильтрацией по категориям (мужские, женские, детские), цене, размеру и поиском по названию;
- сортировать товары по цене и названию;
- добавлять товары в избранное и корзину (требуется авторизация);
- управлять корзиной: изменять количество товаров, выбирать размер, удалять позиции;
- оформлять заказы с демо-страницей оплаты и валидацией банковской карты;
- просматривать личный кабинет с историей заказов, статистикой покупок (общая сумма, количество, любимая категория) и редактировать профиль.

Проект реализован по архитектуре «клиент-сервер»: фронтенд на **Vue 3** связывается с бэкендом на **Go** через **REST API**, данные хранятся в **PostgreSQL**. Безопасность обеспечивается сессионными HttpOnly-куки и хешированием паролей bcrypt.

## Инструкция по запуску (Docker Compose)

Это самый простой способ. Убедитесь, что у вас установлены **Docker** и **Git**.

### Шаг 1. Клонируйте репозиторий

```bash
git clone https://github.com/Nestilabubu/SportStore_Vue_Go.git
cd SportStore_Vue_Go
```

### Шаг 2. Запустите контейнеры

```bash
docker-compose up --build
```

При первом запуске будут скачаны образы, собран фронтенд и бэкенд, поднята база данных. Процесс займёт 2–3 минуты.

### Шаг 3. Откройте приложение в браузере

После успешного запуска в терминале появятся сообщения о старте серверов. Перейдите по адресу:

- **Фронтенд (сайт):** [http://localhost:5173](http://localhost:5173)
- **Бэкенд API:** [http://localhost:8080/api](http://localhost:8080/api) (для проверки работоспособности)

### Шаг 4. Остановка приложения

```bash
docker-compose down
```

## Инструкция по ручному запуску (без Docker)

Если вы хотите запустить компоненты по отдельности.

### 1. База данных PostgreSQL

Установите PostgreSQL локально, создайте базу данных и пользователя. Выполните скрипт `init.sql` (находится в корне проекта) для создания таблиц и наполнения тестовыми товарами.

```bash
# Подключитесь к PostgreSQL и выполните:
psql -U postgres
CREATE DATABASE sportshop;
CREATE USER sportuser WITH PASSWORD 'sportpass';
GRANT ALL PRIVILEGES ON DATABASE sportshop TO sportuser;
\q

# Импортируйте схему и данные
psql -U sportuser -d sportshop -f init.sql
```

### 2. Бэкенд (Go)

```bash
cd backend
go mod tidy
go run main.go
```

Сервер запустится по адресу: `http://localhost:8080`

### 3. Фронтенд (Vue)

Откройте второй терминал.

```bash
cd frontend
npm install
npm run dev
```

Приложение станет доступно по адресу: `http://localhost:5173`

### Важно при ручном запуске:

- Убедитесь, что бэкенд видит вашу базу. При необходимости скорректируйте переменные окружения в файле `backend/.env` (пример см. в разделе ниже).
- Фронтенд должен знать адрес бэкенда: он подхватывает значение `VITE_API_URL` из файла `frontend/.env`. По умолчанию там указано `http://localhost:8080/api`.

## Переменные окружения (если нужно изменить настройки)

### Бэкенд (файл `backend/.env` или переменные в системе)

| Переменная     | Значение по умолчанию | Описание                    |
|----------------|----------------------|-----------------------------|
| DB_HOST        | localhost            | Хост PostgreSQL             |
| DB_PORT        | 5432                 | Порт PostgreSQL             |
| DB_USER        | sportuser            | Имя пользователя БД         |
| DB_PASSWORD    | sportpass            | Пароль пользователя БД      |
| DB_NAME        | sportshop            | Имя базы данных             |
| PORT           | 8080                 | Порт, на котором работает Go |

### Фронтенд (файл `frontend/.env`)

```
VITE_API_URL=http://localhost:8080/api
```



## Коротко: что делает это приложение (для пользователя)

SportShop позволяет:
- ✅ **Смотреть каталог** спортивных костюмов с картинками, ценами и категориями.
- 🔍 **Искать и фильтровать** по названию, размеру, цене и полу (мужские/женские/детские).
- 💖 **Сохранять понравившееся** в избранное.
- 🛒 **Класть товары в корзину**, выбирать размер, менять количество.
- 📦 **Оформлять заказ** с демо-оплатой (валидация карты, но деньги не списываются).
- 👤 **Регистрироваться и входить** в личный кабинет.
- 📊 **Смотреть статистику** трат, любимых категорий и историю заказов.
- ✏️ **Редактировать профиль** (имя, телефон, адрес доставки).
